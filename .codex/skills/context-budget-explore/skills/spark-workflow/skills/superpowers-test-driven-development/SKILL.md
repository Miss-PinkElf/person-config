# Superpowers TDD（内置版）

只在它确实能提升正确性时使用 TDD。

## 最适合的场景

- 纯计算
- 数据格式化或数据变换
- 解析器或映射器逻辑
- 边界条件很多的工具函数

## 循环

1. 为一个行为写一个失败测试
2. 运行并确认失败是预期失败
3. 写最小生产代码让它通过
4. 重跑该测试和相邻受影响测试
5. 只有在测试持续为绿时才重构

## 约束

- 如果你没见过测试先失败，就无法确认它是否真的保护了行为
- 不要超出测试要求过度设计
- 如果主要是 UI 接线或探索性调试，TDD 是可选项，不是强制项
