# Brainstorming 指南

在 SPARK-workflow 的 Align 阶段使用本指南。

## 模式选择

以下情况使用完整 Align：

- 需求存在歧义
- 存在多种可行实现策略
- 会影响多个页面、模块或工作流

以下情况才可使用 Mini Align：

- 改动规模较小
- 预期结果已经足够具体
- 架构和接口边界不会发生实质变化

## 提问规则

- 一次只问一个问题
- 如果决策空间已知，优先给 2-4 个选项
- 在讨论实现细节前，先确认约束、成功标准和非目标
- 不要在同一轮里堆叠多个无关问题

## 方案对比模板

```md
## 方案 A - 推荐
- 为什么适合
- 主要取舍
- 主要风险

## 方案 B
- 为什么也可行
- 为什么当前不作为首选

## 方案 C
- 只有在它与前两个方案有明显差异时才加入
```

## 确认摘要模板

用一个简短摘要，后续可以直接复制进 proposal artifact：

```md
## 已确认方向
- 目标：
- 范围内：
- 范围外：
- 主要实现方向：
- 需要重点关注的风险：
- 仍未解决的开放问题：
```

## 退出条件

只有在用户明确确认方向摘要之后，才能进入 Propose。
