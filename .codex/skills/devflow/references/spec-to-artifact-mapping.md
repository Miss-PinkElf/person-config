# 外部规格到 Artifact 的映射

当用户已经提供 PRD、设计说明、需求文档或其他规格说明时，按以下方式映射到 `devflow` 的正式产物。

## 映射规则

- “为什么做、解决什么问题、业务背景”
  - 写入 `proposal.md`
- “结构、模块边界、接口、数据流、关键技术选择”
  - 写入 `design.md`
- “实施步骤、验收点、拆分任务”
  - 写入 `tasks.md`

## 注意事项

- 不要机械复制原文，先提炼再写入 artifact
- 外部规格有缺口时，要把缺口显式写成开放问题
- 如果规格仍未达成一致，应先回到 Align 或 Plan
