# 轻量路径示例

场景：修改一个现有命令的输出文案与小范围参数校验。

## 路径判断

判断为轻量路径，因为：

- 目标明确
- 影响范围局部
- 不需要完整 proposal/design/tasks

## 执行方式

1. 做一次 Mini Align
2. 写轻量 plan，记录目标、范围、风险
3. 写轻量 tasks
4. 实施
5. 验证
6. 写一个收尾 `checkpoint`
7. 更新 `state.md`

## 需要落盘的内容

- 轻量 plan
- 轻量 tasks
- `checkpoints.md`
- `state.md`

若实施中发现影响范围扩大，再升级为重型路径。
