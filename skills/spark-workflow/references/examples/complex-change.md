# Example: Complex Change

## User Prompt

“Build a new dashboard section with filters, async data loading, empty state handling, and two alternative interaction designs. I am not sure which approach is better yet.”

## Expected Route

Standard path:

```txt
Classify -> Align -> Propose -> Apply -> Review -> Verify -> Archive
```

## Align Outcome

- clarify target users and key actions
- compare at least two layout or interaction approaches
- confirm loading, empty, and error-state expectations

## Artifact Shape

- `proposal.md`: scope, non-goals, edge cases, rollout risk
- `design.md`: page composition, data flow, filter model, API assumptions
- `tasks.md`: scaffold, data layer, UI states, interactions, review, verification
