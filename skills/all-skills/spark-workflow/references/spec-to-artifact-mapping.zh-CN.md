# 规格到 Artifact 的映射

当用户提供 PRD、设计说明、系统分析文档或带注释截图时，使用本指南。

## 映射到 `proposal.md`

需要捕获：

- 业务目标
- 用户价值
- 范围与非目标
- 受影响场景
- 异常与边界场景
- 尚未解决的决策点

建议提问：

- 最低可接受的上线行为是什么？
- 哪些场景被明确排除在范围外？
- 哪些失败态或空状态需要覆盖？

## 映射到 `design.md`

需要捕获：

- 页面与组件结构
- 状态归属
- 数据流
- API 或服务契约
- 复用点
- 性能或迁移风险

建议提问：

- 哪些现有模块可以复用？
- 新状态应该放在哪里？
- 哪些接口或契约可能需要变更？

## 映射到 `tasks.md`

需要捕获：

- 初始化与依赖任务
- UI 任务
- 状态或服务任务
- 测试或验证任务
- 只有在本次变更确实需要时才加入后续清理任务

任务质量规则：

- 每个任务只保留一个主要关注点
- 每个任务都应足够小，便于独立验证
- 避免把实现、调试和验证混成一个模糊的大任务
