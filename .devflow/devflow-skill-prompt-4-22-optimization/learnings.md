# 经验记录（Learnings）

## 2026-04-22

### 文档型 Apply 的验证方式要贴合产物类型

- 场景：本轮变更主要是 `devflow` 技能（DevFlow skill）与 references 文档更新，没有编译或测试入口
- 经验：不要为了“看起来像验证”去伪造构建或测试；更有效的验证证据是关键词落点搜索、正式 diff 检查与跨文档术语一致性核对
- 建议：后续类似技能文档优化任务，优先使用 `rg` + `git diff --check` + 人工通读一致性检查

### Markdown 结构调整时，整文件重写有时比局部补丁更稳

- 场景：`routing-and-stages.md` 在中途中断时出现了“删除后待恢复”的状态
- 经验：对于结构改动较大的 Markdown 文档，如果局部 patch 上下文不稳定，直接按批准设计整文件重写更能避免上下文错位
- 建议：先确认目标结构，再决定是局部 patch 还是整文件替换；中断后优先检查文件是否处于半成品状态
