# Superpowers Brainstorming For SPARK-workflow

Alignment exists to prevent fast wrong implementation.

## Required Behavior

1. Scan relevant code, documents, and patterns first.
2. Ask one question at a time.
3. Prefer multiple-choice questions when practical.
4. Present 2-3 approaches with tradeoffs and a recommendation.
5. Summarize the chosen direction in a form that can become proposal artifacts.
6. Wait for explicit user confirmation before moving on.

Read [../../references/brainstorming-guide.md](../../references/brainstorming-guide.md).

## Outputs

- confirmed constraints
- recommended approach
- remaining open questions
- proposal-ready summary

## Guardrails

- do not skip alignment because the task feels small
- do not jump into implementation
- if the conversation is pure exploration without commitment pressure, route to [../openspec-explore/SKILL.md](../openspec-explore/SKILL.md)
