# Handoff 字段说明

创建 handoff 时，至少覆盖以下字段：

## 当前目标

说明当前 mission 或当前阶段的核心目标是什么。

## 当前阶段

说明当前处于探索、对齐、提案、实施、验证、收尾中的哪一步。

## 当前进度

说明已经推进到什么程度，不要只写“进行中”。

## 本轮完成内容

列出当前会话或当前阶段真正完成的事项。

## 关键决策与原因

不仅写结论，还要写为什么这样选。

## 关键文件 / 产物

指出下一个 agent 或下一次会话必须查看的文件。

## 风险 / 阻塞项 / 开放问题

明确当前未解决事项与潜在风险。

## 立即下一步

必须给出可以直接执行的下一步，不允许只写“继续推进”。

## 恢复指引

告诉下一个执行者该先读什么、再做什么。

## 可从活跃上下文移除的内容

明确哪些推导、讨论、对比过程已沉淀到文件，可以不继续保留在活跃对话中。
