# Superpowers Systematic Debugging（内置版）

没有根因调查，就不要修。

## 必做顺序

1. 先稳定复现失败
2. 仔细读取错误与堆栈
3. 检查最近改动与相关依赖
4. 在组件边界或模块边界收集证据
5. 一次只建立一个假设
6. 先测试最小变更
7. 只有理解根因后，才实施真正修复

## 停止规则

- 如果无法复现，先补数据，不要猜
- 如果两次尝试都失败，放慢节奏并重查假设
- 如果三次尝试仍失败，应质疑设计或架构，并考虑回退到 Propose

## 输出

- 已确认根因
- 证据链
- 最小修复计划
- 修复后的回归证据
